What this page does

Reads and writes Inventory_Control_Workbook.xlsx directly in SharePoint via Microsoft's co-authoring API. Unlimited simultaneous users, no sync conflicts. The workbook remains the system of record.

Daily use — everyone

Search → tap item → tap what happened → qty → Save. Use ＋ New request for things purchasing should order, and ＋ Add item to register a brand-new stocked SKU.

The inbound flow (non-stock requests)

1) Anyone requests an item we don't stock. 2) When the manager orders it, it becomes a real SKU immediately — visible in Items with an INBOUND tag and no location. 3) When it lands on the dock, any signed-in user opens the request (or the item) and taps Mark received — stock is logged automatically. 4) Whoever shelves it taps 📍 Set location. 5) The manager fills in pars/vendor/cost whenever convenient via ✎. Nothing is entered twice; stock only ever changes through the Transaction Log.

Manager tools

Request lifecycle (Acknowledge → Ordered + PO # → Received → Delivered / Cancel), ✎ item editing, one-tap request→SKU conversion, and ⇪ Bulk add: paste rows from Excel or a QuickBooks export (or pick a .csv) to add a whole day's new items at once — a template is downloadable in the dialog. Blank locations are fine; they land as needs-location items.

Adding categories or vendors

Categories live on the workbook's Lists sheet, vendors on the Vendors sheet — the manager adds them there in Excel; the portal picks them up on refresh.

History & search

Tap any item to see its recent transaction history right in the logging panel (expandable to the full trail). The Activity tab now has a search box covering the entire transaction log — filter by SKU, item name, type, initials, job #, or PO. Anyone can set or correct an item's shelf location via the 📍 button; the on-hand dollar value is visible to managers only.

If a save fails

Entries queue locally — Activity tab has Retry. Nothing is lost.

Version

— update procedure and rollback paths are in Feature_Update_Process.docx. SharePoint Version history on the workbook is the universal undo.
